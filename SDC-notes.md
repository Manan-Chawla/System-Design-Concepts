System Design Concept

System

A system is a group of interdependent components linked together according to a plan to achieve a specific objective. A system refers to an organized relationship among functional units or components.

Characteristics of the System

1. Organization :- Also referred to as the arrangement of components.
2. Interaction :- Defined as a way in which components operate with each other
3. Interdependence :- It means how components of a system are dependent on each other.
4. Integration :- It is concerned with how system components are connected together.
5. Central Objective :- The objective of a system has to be central so it can be real or stated.

Pros of System

1. Improve efficency
2. Organized interaction
3. Clear goal achievement

Cons of System

1. Complexity
2. Potential for failure
3. Resistance to change



System Development Approach

It is a structured framework or methodology followed by plan, analyze, design, deploy, and maintain an information system. It provides a systematic and organized way to develop and enhance, ensuring that the entire development process is well defined and follows a logical sequence of activities.

It guides the team through various phases, helping them create a reliable and effective information system.

Pros 

1. Ensure a systematic development process.
2. Provide define framework
3. Optimize use of resources.

Cons

1. Constly
2. Limited flexibility
3. Time Consuming



System Development Process 

It is a series of structured activities or phases that an organization follows to develop, enhance, or maintain an information system. Its aim is to meet the specified requirements and address the needs of the user while considering factors such as time, budget, and quality. 

Elements of System 

1. Input/Output →
Input is information that is entered into the system for processing, and the main aim of the system is to generate output.
2. Processor →
It is the element of the system that involves the actual transformation of input to output.
3. Control →
It is the decision-making subsystem that controls the pattern of activities, its control by OS and software.
4. Feedback →
Provide control in a dynamic system.
5. Enviorment →
It is a “super system” within which an organization operates.
6. Boundaries and Interface →
Boundaries are limits that identify components, while interfaces detect the interaction between these components.
